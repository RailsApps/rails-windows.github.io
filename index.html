Ruby on Rails for Windows

by Joel Dezenzio

Last updated 12 December 2013

Ruby on Rails for Windows is a comprehensive guide to developing web applications using the Ruby on Rails framework on the Microsoft Windows 7 and 8.x operating environments.

Introduction

This guide was created to help developers build Rails applications on the Windows OS platform. If you've searched and combed the web for resources pertaining to Ruby on Rails development on Windows then you've probably realized:

  def information_related_to_windows
    puts ("bangs head on wall " * 1000)
  end

Or, in plain words: It is difficult to find information about developing Rails applications on the Windows platform.

Information is either missing or scattered about in hard-to-find places. You will likely find that Linux and Mac discussions overwhelm your searches on the web. Even if you find some resources that seem relevant, the information may be outdated or unusable. This leads many Windows developers to feelings of frustration. This guide's goal is to provide the information you need to be productive developing with Ruby on Rails on a Windows machine.

  • URU (multi-platform ruby manager)
  • VirtualBox (virtualization application)

You'll also learn about RailsApps which provides Rails Examples and Tutorials.

Misconceptions

As a first step, I'd like to dispel misconceptions that result from a lack of knowledge about Rails for Windows.

Ruby is difficult to install on Windows

You can manually install Ruby using Rubyinstaller and you can also automate installations for Ruby using chocolatey or boxstarter. All of the installation methods are easy and pain-free. Windows installers can also make use of the DevKit which will build and use native C/C++ extensions automatically when required during gem installations.

Windows has no package management

Linux has apt-get for package management. Windows has NuGet, Chocolatey, Boxstarter, and a host of other package management configurations. Chocolatey is the equivalent of apt-get for windows. Boxstarter, coupled with chocolately, allows you to do remote installations, automate installs from the web, network shares, usb keys, or locally. It even has reboot resiliency with administrative rights from start to finish. The NuGet software that helps us to achieve package management is also open source.

Windows has no Shell or Bash

Linux has Linux Shell or Bash. Windows has PowerShell. PowerShell uses cmdlets which allows Windows developers to create complex scripting for managing Windows systems. It makes use of pipes, similar to Linux. However Linux shells output text while PowerShell takes advantage of using objects with the pipeline. It is also used with Chocolatey and package management.

Windows has no RVM

Linux has RVM, the "Ruby Version Manager":https://rvm.io/. Windows can use URU. There are also free Ruby Stack providers like Bitnami which allow developers to deploy a complete environment stack with one-click installation.

The main thing to realize is that, despite misconceptions, Windows is a practical and productive development environment for Rails. Though Rails had its origins in the open source Unix community, there are many talented contributors that invest their time to support development of Rails applications on Windows. The community is friendly and eager to help new developers get up to speed quickly.

What do I need to get started?

  • Patience
  • Some Reading Time
  • Windows 7 (32 or 64-bit), Windows 8 or 8.1 (32 or 64-bit)
  • .NET Framework 4.5 minimum
  • PowerShell 2.0 or higher
  • Fingers (unless you are one of those rare toe-typists)
  • A simple text editor (I recommend Sublime Text 3)

Setting Expectations

When you are finished reading everything (this includes the wiki), you should know how to do the following:

  • How to use Windows as a development platform
  • How to install and use Ruby on Windows via manual or automated methods
  • How to install and use Ruby on Rails via manual or automated methods
  • An understanding of NuGet, Chocolatey, Boxstarter, RubyInstaller, Devkit, Ruby, Ruby on Rails, URU, and PowerShell
  • How to setup and use virtualbox and Windows as a virtual development box
  • How to setup and use git on Windows
  • How to find reference material on the web
  • How to contribute to the discussion
