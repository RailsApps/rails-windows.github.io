Foundation 5

Foundation 5 is a framework designed for responsive layouts on all device types. Whether you are on an iPhone, an Android device, tablets (all sizes), desktops, or even laptops, the goal of foundation is to provide you the best overall view of the site you are visiting.

We've already installed Foundation 5 in Using Ruby on Rails on Windows. Let's dive into our application and start some configuring.

The Basics

If you run the "rails server" command from your wrails application directory and visit http://localhost:3000, you'll see an html page that's pointed to a default "Welcome Aboard. You're riding ruby on rails.". Let's create a simple home page to start developing our layout.

The first thing we'll do is create a controller:

  C:\Development> rails g controller home

We just created some files. Let's go over a few of these together:

  • app/controllers/home_controller.rb - This is our home controller. Rails uses MVC (Model, View, Controller) to handle page requests, database requests, and page views.
  • app/views/home - This is a views directory that will house .html, .html.erb, .js files that are used by our Home Controller.
  • app/helpers/home_helper.rb - This is a helpers file we can use to create methods that will allow us to do different tasks while working in an html template.
  • app/assets/javascripts/home.js.coffee - This is a coffeescript file for our home view.
  • app/assets/stylesheets/home.css.scss - This is a sass stylesheet file.
  • We also have some test files. We won't go over them right now.

Open the app/controllers/home_controller_test.rb file. All it contains is an empty class. Let's add an index method to it. Change your home controller to look like this:

  class HomeController 


Now you'll see we just have an empty index method.  That index method expects that a view with its name exists in the app/views/home directory.  In fact, that
view could be called "index.html" or "index.html.erb".  ERB is known as "embedded ruby" and is part of a templating system in rails that allows us to embed
Ruby code into a text document.  In a web page, it is used to embed ruby into html.  Whenever our app sees an index.html.erb file it understands that
we may have ruby code within, and it will process any ruby code it finds.



From an MVC stand point, we currently have a Controller but we don't have a Model or a View.  Let's make a view.



Create a new file in app/views/home/index.html.erb.  Note the end of my precediing path shows the file index.html.erb.  That will be the file name.
After creating this file, the Home controller's index method now has its expectations of an index view existing met.  Let's add some code to our view.  Add
the following inside of app/views/home/index.html.erb:

We're still missing one piece though, routing.  Open your **app/config/routes.rb" file.  You will see a lot of information has been commented out.  These are
examples of some of the routing conditions you can create as you proceed with your application.  For now, we just want to create a simple pointer for our
application to open the root page to a specific location.  We've created a home index view.  Let's add a route.  Uncomment the line that starts with #root.  See
the code below for an example.



Wrails::Application.routes.draw do
  # The priority is based upon order of creation: first created -> highest priority.
  # See how all your routes lay out with "rake routes".

  # You can have the root of your site routed with "root"
  root 'home#index'




Change your root to root 'home#index'.  This tells our wrails app that our root or default page when people visit our site should be routed to home#index or
controller#view.  Once you save the file, open up your rails app if you haven't done so yet, or simply refresh the page if its open.  If all goes well, you should
see three headings for Our Home Page Exists in... wrails/app/views/home/.
