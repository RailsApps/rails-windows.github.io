Options

This preview chapter will provide you with some install options you have available. Review the options and cases below and best determine your needs. You don't have to make a decision right this minute. This guide houses quite a bit of information and it will take time to comb through its contents.

Installation Options

  • (LOCAL | MANUAL) - Manually install Ruby 2.x and Ruby on Rails (latest) on your local machine
  • (LOCAL | AUTOMATED) - Automatically install Ruby 2.x and Ruby on Rails (latest) on your local machine using Chocalatey, NuGet, and BoxStarter
  • (VM | MANUAL) - Create a Windows 8.1 VM and manually install (Ruby 2.x, Ruby on Rails latest)
  • (VM | AUTOMATED) - Create a Windows 8.1 VM and automatically install Ruby 2.x and Ruby on Rails (latest) using Chocalatey, NuGet, and BoxStarter
  • (BITNAMI) - Create a Windows 8.1 environment with Ruby and Ruby on Rails using Bitnami

Local Machines

Why use Local Machines?

  • You are very familiar with your local workspace and environment.
  • Less prep work may be involved.
  • No additional overhead or hardware resources are required.
  • You can only choose one manual or one automated installation type, not both.

Virtual Machines (VMs)

Why use Virtual Machines?

  • You have an appliance you can create, edit, and destroy without affecting your local system.
  • You can save snapshots and heavily test features.
  • You can try manual or automated installation types as often as you like.
  • Requires additional resources.

Bitnami

Why use Bitnami?

  • Ruby Stacks are already pre-built
  • Has an all-in-one installer for Windows
  • Has a Ruby Stack for VMware
  • Has a Cloud Server Ruby Stack on Amazon, Azure
  • Use this if you just want to get started without doing any type of prep work on your own
